2️⃣ 運用の核となる3つの設定メカニズム

⚙️ 3層のルール定義システム

Claude Codeに一貫性のある行動をとらせるためのレイヤー構造

1

全体ルール設定

プロジェクト全体に適用される共通の行動指針を記述する設定ファイル（claude.md）

内容例：
  • • 報告の形式
  • • 成果物の保存先ルール
  • • 事実と意見の区別などの判断基準
⚠️ 注意点：

文字数は2,000〜3,000文字程度に抑える。
長すぎると指示の優先順位が不明確になるリスクあり

2

フォルダ別ルール

特定の部署（フォルダ）内でのみ適用される個別の指示

例：
  • • CMO: 「集客動線を常に意識する」
  • • CFO: 「精緻な計算と正確な書式を維持する」
💡 メリット：

対象ファイルが限定されるため、
全体ルールよりも詳細な記述が可能

3

専門スキル（Skills）

特定の専門業務に特化した知見をAIに付与する機能

活用例：

「YouTube台本作成スペシャリスト」スキルを作成し、
伸びやすい構成や執筆スタイル、品質基準を記憶させる

✅ メリット：

毎回長いプロンプトを入力する手間を省き、
指示一つで専門家レベルのアウトプットを呼び出せる

3️⃣ 実践的ワークフロー：並列処理と同時進行

⚡ マルチターミナル並列処理

Claude Codeの最大の特徴は、複数のタスクを同時に実行できる点

🖥️ ターミナル左

CMO担当
マーケティングリサーチ
競合分析レポート作成

🖥️ ターミナル中央

CPO担当
営業資料の作成
プレゼンスライド生成

🖥️ ターミナル右

CFO担当
請求書の発行
月次財務レポート

🚀 並列処理のメリット

  • マルチターミナルの活用： VS Code等のターミナルを分割し、複数のClaude Codeを同時に起動
  • 同時並列実行： 左で「マーケティングのリサーチ」、中央で「営業資料の作成」、右で「請求書の発行」
  • 待ち時間の解消： 一つのAIがウェブ検索中に、別のAIへ別タスクを指示できる
  • AI同士の連携： 記事作成担当 → SNS展開担当 → 統合レポート作成、といったリレー形式の業務遂行が可能
