Slidev × Claude 提案資料作成手法

〜Markdown駆動によるデザイン統制と自動変換の統合〜

🔗 「Markdownだけで」顧客提案レベルのスライドを作ってみた【Slidev x Claude Opus 4.6】
エグゼクティブ・サマリー:
「コピペ地獄」や「デザインの不統一」を解消。エンジニアがGitで管理可能なMarkdownを記述し、Claudeが高度なレイアウト（Vueコンポーネント）へ自動変換。ロジック構築に集中できる開発体験（DX）を実現します。

従来手法の課題

  • 複製・差し替えに伴う「コピペ地獄」
  • ページ増大によるデザインのバラつき
  • バイナリ管理によるdiff（差分）確認不可
  • 非効率なレビューと管理の限界

Slidevの選定理由

Viteベースの高速プレビュー Vue 3拡張性 完全テキスト管理 マルチ出力(PDF/Web)

保存と同時に反映されるホットリロードが、執筆のリズムを加速させます。

PptxGenJS との比較

比較軸 PptxGenJS (JS生成) Slidev (Markdown)
コンテンツ記述 API呼び出し Markdown + Vue
デザイン統一 都度スタイル指定 CSS変数で一括管理
LLM親和性 コード生成（誤りあり） テキスト生成（得意領域）

デザインシステムの効果

  • CompareCards: 推奨方式に枠線を自動付与
  • DataTable: ゼブラ柄や強調を自動適用
  • eval-top: 評価基準テーブルを自動描画

導入の利点と制約

◎ 利点: PRベースのレビュー、A4印刷対応
× 制約: 複雑な図解は画像が必要、初期構築コスト
※ 注意: PPTX出力は「画像」扱いとなる

デザインの統制をシステムに委ね、執筆者はロジック構築の本質に集中する。
