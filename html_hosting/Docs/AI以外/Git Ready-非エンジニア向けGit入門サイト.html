📚 Git Ready

非エンジニア向けGit入門サイト
📖 誰もがGitを触る時代に、やさしい入門サイトを作った
📋 エグゼクティブ・サマリー
AI時代の到来により、デザイナーやライター、ディレクターといった非エンジニア層にとってもバージョン管理が不可欠なスキルとなりつつあります。しかし、既存の学習リソースは専門用語やコマンドの羅列に終始し、初心者が抱く「失敗への恐怖」を払拭できていません。

「Git Ready」は、概念図と比喩を先行させた説明、および「失敗した時の戻し方」に重点を置いた構成を採用することで、実務でGitを必要とする人々が「触り続けられる」環境を提供します。

  • ライターの原稿履歴管理

🤖 AIの影響

😰 初心者の恐怖

  • 未知の状態になったときの恐怖
  • 失敗したらどうなるか分からない
  • 元に戻せるか不安

🛡️ Git Readyの解決策

  • 心理的安全性の確保
  • 「戻し方」を必ず説明
  • 概念を先に図解で理解
  • 日常語での言い換え
🎨 4つの設計方針

📊 概念先行型のアプローチ

  • コマンドの前に図解や比喩で説明
  • 専門用語は日常語で言い換える
  • 「ブランチ」→「木の枝」「平行宇宙」

🛡️ 心理的安全性の確保

  • 「元に戻せる確信」が必要
  • 各章の最後に「戻し方」セクション
  • 失敗を恐れず触り続けられる

🎯 学習目標の最適化

  • エンジニアレベルを目指さない
  • 実務で困らない最低限の知識
  • まず動かせる状態に
💡 開発プロセスにおける洞察

🔍 言語化による理解の深化

専門用語を日常的な概念へ翻訳する作業（例：「ブランチ」を「木の枝」や「履歴の平行宇宙」と表現）は、教える側の理解の曖昧さを浮き彫りにし、知識の精度を高める。

⚖️ 適切な距離感の維持

全てを完璧に教えようとすることは初心者の学習を停滞させる。実務との適切な距離感を保った情報提供が重要。

💬 重要メッセージ
初心者が安心して触り続けるためには「元に戻せる確信」が必要
実務で困らない「最低限の知識」を確実に習得。まずは動かせる状態に引き上げる
🎓 結論
Gitが一部の専門家だけの道具であった時代は終焉を迎えつつある

非エンジニアがバージョン管理という新たな領域に踏み出す際、その心理的・技術的な障壁を低減するための「やさしい階段」としての役割が、今後の学習プラットフォームには求められている。
