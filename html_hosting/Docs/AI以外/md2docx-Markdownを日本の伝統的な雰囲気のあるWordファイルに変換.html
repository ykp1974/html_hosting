📖 Markdownを日本の伝統的な雰囲気のあるWordファイルに変換するCLIツール「md2docx」を作った
🎯 概要
md2docxは、Markdownファイルを日本のビジネス文書の慣習に適合したWord（.docx）形式へ変換するためのRust製CLIツールです。エンジニアが使い慣れたMarkdownでドキュメントを作成しながら、クライアントや業務で求められる「日本の伝統的な雰囲気」を持つWordファイルを一発で生成することを目的に開発されました。
💡 開発背景と解決する課題

❌ 課題

  • エンジニアはMarkdownを好む
  • 実務ではWord形式が必須
  • 既存ツールでは日本語特有の書式に対応困難
  • 図表番号の自動採番ができない
  • 手動修正コストが高い

✅ 解決策

  • Markdownの生産性を維持
  • 日本のビジネス文書基準に対応
  • 游明朝・游ゴシックをデフォルト適用
  • 図表番号を自動採番
  • TOML設定で一括カスタマイズ
⚡ 主な特徴と機能

🛠️ 導入と利用方法
必要要件: Rust 1.70以上

基本コマンド

mdd input.md

オプション指定

mdd input.md -o output.docx -c config.toml
オプション 説明
-o, --output 出力ファイル名を指定
-c, --config 設定ファイルを指定

🎓 結論
md2docxは、Markdownの生産性とWordのフォーマット要求を両立させるツールです。特に「図表番号の自動採番」という従来のツールでは困難だった課題を解決しており、実務レベルで「日本の伝統的な業務」に耐えうるドキュメント作成を強力に支援します。
