3. 重大事例：プロでも防げない「1行のミス」

設定ファイル .npmignore のわずか1行の書き忘れにより、51万行超のソースコードが流出。4万件以上フォークされ回収不能に。

スロックスクワッティング

AIの「幻覚」が提案した存在しないパッケージ名を攻撃者が先取り登録。ユーザーが誤って悪意あるコードを導入する新型攻撃。

4. プロフェッショナルが実践する防衛術

パスワードマネージャー + MFA フィッシング自動入力防止と、漏洩時の「最後の砦」を構築。
最小権限の原則 (Least Privilege) AIには必要最小限のフォルダ・APIのみ露出。秘密情報は隔離する。
多層防御 (Defense in Depth) 100点の壁はない前提で、不完全な対策を何層にも重ねる。
職務分離：AIはインターン 「申請者（AI）」と「承認者（人間）」を明確に分け、最終承認は必ず人間が行う。
攻撃面（窓）を減らし、被害範囲（延焼）を最小化せよ。

秘密情報の隔離、公式リポジトリの確認、そして「迷ったらNo」がAI時代の新鉄則です。
